## RPA浏览器控制台 WebRTC 集成与 API 修复（第二轮）

### 发现的问题
- `RealTimeControlModal.vue` 调用了大量已删除 API：`getScreenshot`、`navigateToUrl`、`executeJavaScript`、`safeExecuteJavaScript`、`clickElement`
- `BrowserDebugPanel` 没有集成 WebRTC 实时视频流功能

### 修复和新增
1. **`RealTimeControlModal.vue`**：修复所有已删除 API 调用，改用 `executeAction` 接口
   - `takeScreenshot` → `executeAction({ action_id: 'screenshot' })`
   - `handleNavigate` → `executeAction({ action_id: 'navigate' })`
   - `executeJavaScript/safeExecuteJavaScript` → `executeAction({ action_id: 'execute_javascript' })`
   - `performClick` → `executeAction({ action_id: 'click' })`
   - 移除 `blobToBase64` 辅助函数（不再使用）

2. **新增 `WebRTCStreamPanel.vue`**：从 `RealTimeControlModal` 提取核心 WebRTC 逻辑
   - 内嵌于 Tab 中（非弹窗），视频+控制面板左右布局
   - 功能：WebRTC 连接/断开/重连、截图、URL 导航、JS 执行、坐标点击
   - 日志区实时记录操作状态

3. **`BrowserDebugPanel.vue`**：新增"实时控制" Tab（默认激活），集成 `WebRTCStreamPanel`

4. **`VisualControlPanel.vue`**：新增"实时视频"快捷入口卡片，点击切换到实时控制 Tab

### WebRTC 流程
1. 调用 `/api/v1/rpa/browser/control/webrtc/offer` 获取服务端 SDP offer
2. 创建 `RTCPeerConnection`，设置远程描述
3. 获取服务端 ICE candidates 并添加
4. 创建 answer，等待 ICE gathering 完成
5. 发送 answer 到 `/api/v1/rpa/browser/control/webrtc/answer`
6. 视频流通过 `ontrack` 回调绑定到 `<video>` 元素
