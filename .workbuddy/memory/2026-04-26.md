## BrowserDebugPanel Tab 空白修复

**问题**：实时控制、可视化操作、自定义操作、Debug 调试四个 tab 打开全是空白。

**根因**：`BrowserDebugPanel.vue` 的 CSS 问题：
1. `el-tabs__content` 设置了 `overflow: hidden`，但未设置 `el-tab-pane` 高度为 100%，导致内容高度超出被裁掉显示为空白

**修复**：`src/components/browser/BrowserDebugPanel.vue`
- 四个 tab 的包装 div 统一加 `overflow-y-auto` class
- CSS：`el-tab-pane` 设置 `height: 100%`，`tab-content` 设置 `height: 100%; overflow-y: auto`

## FingerprintList quick-edit 事件转发 Bug 修复

**问题**：`rename_fingerprint` 请求体始终为 `{}`，后端收不到参数。

**根因**：`FingerprintList.vue` 第 120 行：
```html
@quick-edit="$emit('quick-edit', browserId, newName)"
```
`browserId` 和 `newName` 在 `FingerprintList.vue` 作用域中不存在（它们是 `FingerprintCard.vue` 内部变量），导致 emit 了 `undefined, undefined`，请求体序列化为 `{}`。

**修复**：改为 rest 参数透传子组件参数：
```html
@quick-edit="(...args) => $emit('quick-edit', ...args)"
```

## API 错误提示增强 — 业务失败分支补充请求信息

**问题**：FastAPI 422 参数校验错误经 Node.js 代理包装后变成 `code !== 0` 的业务响应（走 success 回调的 `data.code !== 0` 分支），之前只在 error 回调（网络错误）中拼接了请求 URL/body，业务失败分支的 `response.msg` 没有请求信息。

**修复**：
1. `base_axios.ts` 响应拦截器 success 回调：`data.code !== 0` 时，将请求 method、URL、params、body 拼接到 `data.msg` 尾部
2. `businessHandler.ts` catch 分支：优先使用 error.msg（axios error 回调已格式化的，含请求信息），避免被 errorMessage 覆盖导致请求详情丢失

## 控制台子路由 browserId 未传递导致 API 请求 body 为 {}

**问题**：控制台子路由组件（WebRTCStreamPanel、VisualControlPanel、CustomActionPanel、DebugPanel）通过 `defineProps<{ browserId: string }>()` 接收 browserId，但路由配置的子路由没有设置 `props`，导致 `<router-view />` 不会将路由参数作为 prop 传递，`props.browserId` 为 `undefined`，API 请求 body 中 `{ browser_id: undefined }` 被 JSON 序列化为 `{}`。

**修复**：`src/router/index.ts` — 为四个子路由都添加 `props: (route) => ({ browserId: route.params.browserId })`，将父路由的 `:browserId` 参数通过 props 传给子组件。

## 控制台子路由切换去除"正在前往"loading

**问题**：控制台面板内 tab 切换（webrtc ↔ visual ↔ custom ↔ debug）是同一父路由下的子路由切换，但全局路由守卫仍会 emit loading 显示"正在前往：【xxx】"，体验不好。

**修复**：`src/router/index.ts` 路由守卫中增加判断——当 `from.matched[0] === to.matched[0]`（同一父路由的子路由切换）时，跳过 loading 遮罩。

## 启动/终止按钮重构为前端状态机

**需求**：点击启动后轮询设上限，超时报错；启动中显示加载样式和"启动中..."，失败显示"启动失败"；整体写为状态机。

**实现**：`BrowserConsolePanelView.vue`
- 新增 `OperateState` 类型：`'idle' | 'launching' | 'running' | 'launch_failed' | 'stopping'`
- 启动按钮：idle→显示"启动"（primary），launching→显示"启动中..."（loading+disabled），launch_failed→显示"启动失败"（danger，可重试）
- 终止按钮：仅在 running 状态显示，stopping 时 loading+disabled
- 启动后开启 2s 间隔轮询（最多 30 次 = 60s），检测到 `running` 则成功，检测到 `error` 或超限则报错并切到 `launch_failed`
- 终止后递归检查（最多 10 次 × 2s），后端确认停止后回到 idle
- 常规 5s 轮询同步后端状态到状态机（不在 launching/stopping 时覆盖）
